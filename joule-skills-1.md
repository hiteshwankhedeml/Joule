# Joule Skills

* Execute repetitive tasks to improve efficiency
* Performs one predefined action
* So that employees don't have to juggle multiple skills



* When a Skill evolves—perhaps because an extra parameter is needed—developers release version 1.1;&#x20;
* if the data schema must break, they publish version 2.0 and leave the earlier branch untouched until all consumers have migrated.&#x20;
* Telemetry on latency and error codes highlights Skills that need optimisation or refactoring, and life-cycle tooling lets you freeze or retire older versions once they are no longer required.



**Procedure:**

1. Name your skill

* Write a one-sentence purpose statement for the intent directory

2. Configure inputs & outputs
3. Embed Automations
4. Add actions and messages
5. Test and refine
6.  Deploy
